2.1长期记忆刻画人物画像：每次对话调用工具分析人物画像
    人物画像：基础病+急性病+过敏史+用药史+手术史+家族史+生活习惯+职业暴露+旅行史+疫苗接种史+心理健康史+社会经济状况+文化背景+语言偏好+教育水平+信息素养+对医疗的态度+对风险的承受能力+对治疗的期望+对医生的信任度+对医疗系统的了解程度+对医疗资源的获取能力+对医疗费用的支付能力+对医疗服务的需求程度+对医疗服务的满意度+对医疗服务的期望
    这个轻量级长期记忆（用户画像）方案的核心思想是：“摒弃向量检索，采用JSON状态机 + 工具调用”。

    以下是该方案的高度精简总结：

    核心运作机制（三步走）
    极简存储（存）：抛弃笨重的向量数据库，改用一个轻量的 JSON 文件或字段（如 user_profile.json），结构化地存储用户的“基本信息”、“偏好习惯”和“当前语境”。
    
    动态前置（读）：在每次对话时，将这个 JSON 文件提取为自然语言文本，无缝拼接到 System Prompt 的头部，让 Agent 随时“带着记忆聊天”。
    
    自主进化（写）：为 Agent 提供一个 Function Calling 工具（如 update_profile）。当 Agent 在对话中察觉到用户的新习惯或信息变更时，自主调用该工具去修改底层的 JSON 文件。
